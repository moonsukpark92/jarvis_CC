## 0. 개요 (Executive Summary)

### 비전
> "데비스의 실체(Brain)는 집 노트북에 두되, 박대표님 곁(Edge)에는 항상 그의 귀와 눈을 둔다."

데비스는 현재 데스크톱에 상주하는 음성 AI 집사입니다. 본 계획은 데비스를 **삼성 스마트폰 앱**으로 확장하여, 박대표님의 일상(회의·미팅·이동·업무)을 24/7 청취·기록·분석하고, 학습된 인사이트를 자율 보고하는 **개인 AI 부관(Adjutant)** 으로 진화시키는 것을 목표로 합니다.

### 핵심 원칙
1. **Hub-Spoke 아키텍처**: 집 노트북 = 두뇌(Hub), 스마트폰 = 감각기관·말단부(Spoke)
2. **프라이버시 우선**: 녹음·전사·분석은 박대표님 개인 인프라 안에서만 처리 (외부 서드파티 클라우드 ASR/LLM 사용 시 명시적 동의)
3. **상시 가용성**: 인터넷이 끊겨도 로컬 캐시·온디바이스 STT로 최소 동작 유지
4. **자가 학습**: 매일 자정 일과 요약 → 특이점 추출 → 박대표 페르소나 모델 업데이트

### 결과물 한 줄 정의
**"박대표님이 회의실에 들어가 폰을 책상에 놓는 순간, 데비스가 듣고 분석하고 기록하며, 회의가 끝나면 요약·액션 아이템·특이점 리포트가 텔레그램으로 도착한다."**

## 2. 모바일 앱 (Android, Samsung 최적화)

### 2.1 기술 스택

- **언어**: Kotlin + Jetpack Compose
- **최소 SDK**: API 34 (Android 14) — Z Flip5 출하 OS 기준
- **타겟**: API 35 (Android 15) — One UI 7 업그레이드 대비
- **빌드**: Android Studio Hedgehog+, Gradle 8.x
- **DI**: Hilt
- **비동기**: Kotlin Coroutines + Flow
- **네트워크**: Ktor Client (WebSocket), Retrofit (REST)
- **로컬 DB**: Room (대화·세션 메타데이터)
- **암호화**: Android Keystore + EncryptedSharedPreferences
- **NPU 가속**: Qualcomm QNN SDK (Snapdragon 8 Gen 2 Hexagon NPU) — whisper.cpp INT8 양자화 모델 추론 2-3배 가속
- **Flex Window UI**: Samsung Good Lock "MultiStar" / Cover Screen API (Galaxy SDK)

### 2.2 핵심 구성요소

#### (a) Foreground Service — `DavisListenerService`
- 상시 동작하는 포그라운드 서비스 (배터리 최적화 예외 등록 안내)
- 알림 채널: "데비스 청취 중" + 일시정지 액션
- `MediaProjection` 또는 `AudioRecord` 로 PCM 스트림
- **Android 15 부분 화면 공유 API** 활용 가능 (회의 앱 오디오 캡처)

#### (b) On-device VAD/STT
- **Silero VAD ONNX** (~2MB, <5ms 추론) — 침묵 구간 자동 컷
- **whisper.cpp** Android NDK 빌드, `ggml-tiny-q5_1.bin` (40MB, 한국어 동작 가능)
- 폰 NPU(삼성 NPU SDK) 활용 시 성능 2-3배 향상 검토 (Galaxy S24+ 이상)

#### (c) 트리거 메커니즘 (Z Flip5 최적화)

| 트리거 | 방법 | 시나리오 |
|--------|------|----------|
| **음성 웨이크워드** | "데비스" + Porcupine 커스텀 | 핸즈프리 |
| **빅스비 루틴** | "회의실 Wi-Fi 연결됨 → 데비스 시작" | 자동 |
| **캘린더 연동** | Google Calendar 회의 5분 전 자동 활성화 | 자동 |
| **Flex Window 단축 버튼** | 커버 화면 위젯 더블탭 (폰 접은 상태로 책상 거치) | 수동 |
| **폴드 제스처** | "회의 중 절반 접기(Flex Mode) → 자동 청취 시작" | 자동 |
| **사이드 키 두 번 누르기** | 빅스비 루틴 트리거 (S Pen 대체) | 수동 |
| **위젯/홈 단축키** | 메인 화면 4×1 위젯 | 수동 |
| **NFC 태그** | 회의실 책상 NFC 태깅 | 자동 |
| **갤럭시 워치 컴패니언** | 손목에서 즉시 트리거/취소 | 수동 |

#### (d) 통신 클라이언트
- WebSocket Secure (WSS) 영속 연결
- 메시지 프로토콜: Protocol Buffers (효율) 또는 JSON (디버깅)
- 재연결 지수 백오프 (최대 60초)
- 오프라인 큐: Room DB에 chunk 저장 → 재연결 시 일괄 전송

### 2.3 UI 화면 구성

1. **홈** — 현재 세션 상태, 오늘 분석된 회의 수, 다음 일정
2. **세션 라이브** — 실시간 전사 스트림 + 화자 분리 + 키워드 하이라이트
3. **타임라인** — 일자별 회의·통화·메모 카드
4. **리포트** — 일일/주간/월간 요약, 특이점, 액션 아이템 진행률
5. **데비스 음성 채팅** — 기존 데스크톱과 동일 UX (저지연 음성 대화)
6. **설정** — 트리거·프라이버시·동기화·웨이크워드 학습

### 2.4 삼성 특화 통합 (Z Flip5 전용 항목 포함)

#### Z Flip5 Flex Window (커버 3.4") 활용
- **상시 청취 인디케이터**: 커버 화면 좌상단 작은 점 + "데비스 청취 중" 텍스트
- **실시간 전사 티커**: 커버 화면에서 마지막 발화 한 줄 스크롤 (회의 중 책상 위 거치)
- **즉시 액션 카드**: 데비스 분석 결과 ("거래처 K가 이번 주 답변 예정") 커버에 푸시
- **퀵 답변**: 데비스 능동 학습 질문에 커버에서 👍/👎 즉답 (Wear OS Tile API와 유사 패턴)
- **Always-On Display**: 다음 회의 5분 전 카운트다운 + 자동 활성화 안내

#### Flex Mode (반접기) 활용
- **회의실 책상 거치 모드**: 폰을 ㄴ자로 세워 두면 자동 감지 → 청취 시작
- **상단 화면**: 데비스 상태/실시간 전사
- **하단 화면**: 빠른 컨트롤 (일시정지·종료·메모 추가)

#### 일반 삼성 통합
- **Samsung Health 연동**: 심박/스트레스 → 회의 강도 상관분석
- **Samsung Notes**: 회의 요약 자동 저장 (One UI 7 AI 메뉴 통합 시점에 강화)
- **갤럭시 워치 컴패니언**: 손목에서 즉시 트리거/취소 (Wear OS 4+)
- **One UI Edge Panel**: 데비스 빠른 패널 (실시간 전사 미리보기)
- **빅스비 루틴 SDK**: 회의실 Wi-Fi/블루투스/NFC/시간대 조건 → 자동 액션

> **제외 항목 (Z Flip5 미지원)**: S Pen Air Action, Samsung DeX 데스크톱 대시보드

## 3. 백엔드 (Hub) 확장

### 3.1 신규 모듈

#### (a) `sync_server.py` — FastAPI + Uvicorn
- WebSocket: `/ws/audio` (모바일 → 허브 음성 chunk 스트리밍)
- WebSocket: `/ws/control` (양방향 명령·상태)
- REST: `/api/sessions`, `/api/reports`, `/api/memory/search`
- 인증: 사전 발급 디바이스 토큰 + mTLS 클라이언트 인증서

#### (b) `meeting_processor.py`
- VAD 기반 세션 자동 분할 (5분 이상 침묵 → 세션 종료)
- **화자 분리**: `pyannote-audio` (3.x) 또는 `Resemblyzer` 임베딩
- **고품질 STT**: `faster-whisper large-v3` 또는 한국어 FT 모델
- 정렬: 화자별 타임라인 발화 시퀀스 생성

#### (c) `meeting_analyzer.py` (Claude 기반)
입력: 회의 전사 + 화자별 발언 + 캘린더 메타
출력 스키마:
```json
{
  "summary": "5문장 이내 핵심 요약",
  "decisions": ["결정 사항 리스트"],
  "action_items": [
    {"owner": "박대표", "task": "...", "due": "2026-04-30", "priority": "high"}
  ],
  "anomalies": [
    {"type": "감정", "detail": "거래처 K가 평소보다 강한 어조", "evidence": "..."}
  ],
  "keywords": ["가격 협상", "납기", "...
"],
  "follow_up_questions": ["..."]
}
```

#### (d) `report_generator.py`
- **일일 브리핑** (매일 22:00): 오늘의 회의 N건 → 통합 요약 + 내일 준비사항
- **주간 리포트** (일요일 21:00): 주간 트렌드, 반복되는 이슈, KPI
- **즉시 리포트**: 회의 종료 5분 후 텔레그램 푸시
- 출력: 마크다운 → Google Docs 자동 저장 + 텔레그램 메시지 + 데비스 음성 브리핑

#### (e) `learning_loop.py` (자가 학습)
- 매일 자정: 신규 발언·결정·액션을 벡터 DB에 인덱싱
- 메모리 사실 자동 추출: "거래처 K는 가격에 민감함" 같은 패턴화
- 페르소나 캘리브레이션: 박대표의 말투·결정 스타일 통계 → 시스템 프롬프트 미세조정 후보 생성 (적용은 박대표 승인 필수)
- **벡터 DB**: ChromaDB 또는 Qdrant (로컬 임베딩, 외부 송신 없음)

### 3.2 스토리지 레이아웃

```
~/.jarvis-cc/
├── audio/           # 원본 음성 (회의별 폴더, AES-256 암호화)
├── transcripts/     # 전사 JSON (화자 분리 포함)
├── reports/         # 일일/주간/회의별 리포트 마크다운
├── vectors/         # ChromaDB 영속 디렉터리
├── memory/          # 기존 facts.json + 신규 persona_profile.json
├── sessions.db      # SQLite (세션 메타·인덱스)
└── sync.db          # 모바일 동기화 상태
```

## 4. 보안 및 프라이버시

### 4.1 위협 모델
- 분실/도난 폰 → 음성·전사 유출
- 가정 네트워크 침입 → 허브 접근
- 악성 앱 → 마이크 도청
- 클라우드 서드파티 → 데이터 누출

### 4.2 대응

| 항목 | 조치 |
|------|------|
| 폰 저장 데이터 | Android Keystore + AES-256 (지문/홍채 잠금 시에만 복호화) |
| 통신 | TLS 1.3 + mTLS, 인증서 핀닝 |
| 허브 접근 | Tailscale 메시 VPN (외부 노출 0), 또는 Cloudflare Tunnel |
| 음성 백업 | 클라이언트 사이드 암호화 후 Google Drive (키는 허브에만) |
| LLM 프롬프트 | 민감 키워드 자동 마스킹 옵션 (이름/금액/계좌) |
| 녹음 동의 | 회의 시작 시 "녹음 중" 음성 안내 (한국 통신비밀보호법 대응 — 본인 참여 대화는 합법) |
| 삭제권 | 30/60/90일 자동 만료 정책 + 즉시 폐기 명령 |

### 4.3 법적 고려사항
- 한국: 본인이 참여한 대화 녹음은 합법 (통비법 §3①). 단, 제3자 간 대화 도청은 불법
- **앱 내 명시 동의 화면**: "본 앱은 박대표님이 참여한 회의·통화만 녹음하며, 제3자 간 대화 녹음을 위해 사용해서는 안 됩니다"
- GDPR/PIPA: 박대표 개인 사용이라 개인정보처리자 대상 아님 (단, 회의 참석자 발언 보관 시 내부 정책 명시 권장)

## 5. 자가 학습 시스템 상세

### 5.1 학습 대상
1. **박대표 페르소나**: 결정 패턴, 선호 표현, 우선순위 가치
2. **인물 프로파일**: 거래처/임직원별 성향·핫버튼·이력
3. **업무 패턴**: 시간대별 업무 강도, 회의 빈도, 반복 의제
4. **특이점 임계값**: "정상" 기준선을 학습해 이상 신호 자동 감지

### 5.2 학습 파이프라인

```
[일일 회의 N건]
    ↓
[Claude Sonnet 추출] ─────┐
    │                     ↓
    │              [구조화 데이터]
    │              {persons, decisions, anomalies}
    ↓                     ↓
[원본 텍스트 임베딩]   [팩트 후보 생성]
    ↓                     ↓
[ChromaDB 인덱싱]    [중복/모순 검사]
                          ↓
                   [박대표 검토 큐] ←── 텔레그램으로 "이거 맞나요?" 확인
                          ↓ (승인)
                   [memory.facts 업데이트]
                          ↓
                   [persona_profile.json 갱신]
```

### 5.3 능동 학습 (Active Learning)
- 데비스가 불확실한 항목은 즉시 추정하지 않고, 박대표에게 텔레그램/음성으로 질문
- 예: "오늘 김상무님이 '검토하겠다'고 하셨는데, 보통 이건 긍정 신호인가요 아니면 보류 신호인가요?"
- 답변은 라벨로 저장 → 다음 분석 정확도 향상

### 5.4 자기 평가 (Self-Eval)
- 주간 1회: 지난주 예측 vs 실제 결과 비교
- 예: "월요일에 '거래처 K는 이번 주 답을 줄 것'으로 예측 → 실제로 목요일에 회신" → 패턴 강화
- 메트릭: 액션 아이템 완료율, 특이점 적중률, 리포트 박대표 평점(👍/👎)

## 7. KPI 및 성공 기준

| 지표 | 목표 | 측정법 |
|------|------|--------|
| 음성 → 첫 전사 지연 | ≤ 3초 | 폰 → 라이브 화면 |
| 회의 종료 → 리포트 도착 | ≤ 5분 | 회의 30분 가정 |
| STT 정확도 (한국어) | WER ≤ 12% | 테스트 셋 100문장 |
| 화자 분리 정확도 | DER ≤ 15% | 3인 회의 기준 |
| 액션 아이템 누락율 | ≤ 10% | 박대표 사후 점검 |
| 배터리 (8h 청취 + 4h 대화) | ≤ 30% 소모 | **갤럭시 Z Flip5 (3,700 mAh)** 기준, 절전 듀티사이클 적용 |
| 배터리 (24h 상시 대기) | ≤ 18% 소모 | 웨이크워드만 활성, NPU 슬립 |
| 박대표 만족도 | 주간 ≥ 8/10 | 주간 1회 자기 평가 |

## 8. 리스크 및 대응

| 리스크 | 영향 | 대응 |
|--------|------|------|
| 한국어 STT 정확도 부족 | 분석 품질 저하 | 한국어 FT Whisper + 도메인 어휘 사전 |
| 배터리 과소비 | 사용성 ↓ | NPU 활용 + 듀티 사이클링 + 빅스비 루틴 |
| 가정 인터넷 단절 | 분석 중단 | 폰 로컬 캐시 + 4G 폴백 + 야간 일괄 동기화 |
| 회의 동석자 거부감 | 사회적 이슈 | "녹음 중" 음성 고지 + 본인 발언만 옵션 |
| 클라우드 LLM 비용 | 운영비 증가 | Haiku 기본 + Sonnet은 분석 시만 + 프롬프트 캐싱 |
| 모델 환각 | 잘못된 보고 | 모든 액션 아이템에 원문 인용 첨부 + 박대표 승인 후 캘린더 등록 |
